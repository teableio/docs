title: "记录更新时"
description: "当已有记录被修改时触发工作流"
---

<Tip>告诉 AI 你希望工作流做什么，AI 会自动选择合适的触发器并为你搭建工作流。</Tip>

记录被修改时触发，新建时不会触发。

## 用 AI 创建

打开表格右侧边栏的 AI 对话，描述你的需求。

AI 会为你处理一切：选择合适的触发器，映射相关字段，自动设置所有操作。

只需描述一次目标，工作流即刻就绪，无需手动配置。

**示例：** *"状态字段变更时给负责人发邮件"*

## 配置

| 设置 | 必填 | 说明 |
|---|---|---|
| 表格 | 是 | 要监控的表格 |
| 监听字段 | 是 | 选择一个或多个特定字段，或选择"全部字段" |
| 筛选条件 | 否 | 仅在更新后的记录满足这些条件时触发 |

## 如何设置

1. 新建工作流，选择**记录更新时**作为触发器。
2. 选择要监控的表格。
3. 配置监听字段——建议选择你真正关心的字段，而非"全部字段"。
4. （可选）添加筛选条件来进一步限定触发范围。
5. 添加后续操作步骤。
6. 点击**测试**验证触发器配置。
7. 开启工作流。

## 监听字段详解

监听字段决定了哪些字段的变化会触发工作流：

- **选择特定字段**：只有当你选定的字段发生变化时才触发。例如只监听"状态"字段，那么修改"备注"字段不会触发工作流。
- **全部字段**：任何字段的变化都会触发，包括自动计算字段、关联字段等。

<Tip>强烈建议选择特定的监听字段。选择"全部字段"容易导致工作流在不相关的编辑中被频繁触发，浪费运行次数，也可能导致意外行为。</Tip>

## 筛选条件说明

筛选条件在记录更新后被评估。只有当更新后的记录满足筛选条件时，后续步骤才会执行。常见用法：

- 只在状态变为"已完成"时触发后续操作
- 只处理特定部门或分类下的记录变更

## 触发后可用的数据

触发后，后续步骤可以访问该记录的所有字段数据（更新后的最新值），包括记录 ID。你可以在后续步骤中通过点击 **+** 按钮来引用这些数据。

## 适用场景

- **价格同步**：当产品表中的价格字段被修改时，自动通过 HTTP 请求将新价格同步到外部电商平台或 ERP 系统，确保各渠道价格一致。
- **任务状态通知**：监听任务表的"状态"字段，当任务状态变为"已完成"时，自动发送邮件通知项目负责人，同时在日志表中记录完成时间。
- **变更审计日志**：监听关键数据表的重要字段，每次变更时自动在审计表中创建一条记录，写入变更时间和变更人信息，方便后续追溯。

## 常见问题

### 如何避免更新循环？

<Warning>如果工作流中包含"更新记录"操作，且更新的是同一个表的同一条记录的被监听字段，会导致工作流再次触发，形成无限循环。</Warning>

避免方法：
- 在"更新记录"操作中，只更新不在监听范围内的字段
- 使用筛选条件限制触发范围，例如"状态不等于已处理"
- 将需要被自动化修改的字段（如时间戳）排除在监听字段之外

### 批量更新会怎样？

通过 API 或批量操作修改多条记录时，每条记录的更新都会单独触发工作流。

### 与"记录满足条件时"有什么区别？

"记录更新时"在每次监听字段变化时都会触发；"记录满足条件时"只在状态从"不满足"变为"满足"时触发一次。如果你只需要在首次达到某个条件时执行操作，建议使用[记录满足条件时](/zh/basic/automation/trigger/records/record-matches-conditions)。
