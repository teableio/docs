---
title: "配置 SMTP 和发件人"
description: "配置自定义 SMTP 服务器和发件人信息用于自动化邮件发送"
---

默认情况下，自动化使用 Teable 内置的邮件服务发送邮件。如果你需要使用自己的企业邮箱地址发送，或者对邮件送达率有更高要求，可以配置自定义 SMTP 服务器。

## 为什么使用自定义 SMTP

| 场景 | 说明 |
|---|---|
| 企业品牌形象 | 从公司域名发送邮件（如 `noreply@yourcompany.com`），而非 Teable 默认地址 |
| 提高送达率 | 自有域名配合 SPF/DKIM 记录，减少邮件被标记为垃圾邮件的概率 |
| 合规要求 | 部分行业要求邮件必须从指定域名发送，留有可追溯的记录 |
| 已有邮件基础设施 | 复用公司现有的邮件服务器或第三方发送服务（如 Amazon SES、SendGrid） |

## 配置

| 设置 | 必填 | 说明 |
|---|---|---|
| SMTP 服务器 | 是 | 邮件服务器地址（如 `smtp.gmail.com`） |
| 端口 | 是 | 通常为 `465`（SSL）或 `587`（TLS） |
| 用户名 | 是 | SMTP 账户的用户名，通常是邮箱地址 |
| 密码 | 是 | SMTP 账户密码或应用专用密码 |
| 发件人名称 | 否 | 收件人看到的发件人显示名称 |
| 发件人地址 | 是 | 发件人邮箱地址 |

## 如何设置

1. 前往**空间 → 设置 → 邮件设置**。
2. 填写 SMTP 服务器信息（见上表）。
3. 点击**测试连接**验证配置是否正确。
4. 保存设置。
5. 配置完成后，该空间下所有使用"发送邮件"操作的自动化都会通过你的 SMTP 服务器发送。

## 常见 SMTP 服务器配置

| 服务商 | 服务器地址 | 端口 | 备注 |
|---|---|---|---|
| Gmail | `smtp.gmail.com` | 465 (SSL) 或 587 (TLS) | 需使用应用专用密码 |
| Outlook / Microsoft 365 | `smtp.office365.com` | 587 (TLS) | 可能需要管理员启用 SMTP AUTH |
| QQ 邮箱 | `smtp.qq.com` | 465 (SSL) | 需开启 SMTP 服务并获取授权码 |
| 163 邮箱 | `smtp.163.com` | 465 (SSL) | 需开启 SMTP 服务并获取授权码 |
| 阿里企业邮箱 | `smtp.qiye.aliyun.com` | 465 (SSL) | 使用企业邮箱账户和密码 |
| 腾讯企业邮箱 | `smtp.exmail.qq.com` | 465 (SSL) | 使用企业邮箱账户和密码 |
| Amazon SES | `email-smtp.{region}.amazonaws.com` | 465 (SSL) | 需创建 SMTP 凭证（非 AWS 账户密码） |
| SendGrid | `smtp.sendgrid.net` | 587 (TLS) | 用户名固定为 `apikey`，密码为 API Key |

## 常见错误和解决方法

| 错误 | 原因 | 解决方法 |
|---|---|---|
| 连接超时 | 服务器地址或端口错误 | 检查服务器地址拼写和端口号。465 对应 SSL，587 对应 TLS，不要混用 |
| 认证失败 | 用户名或密码错误 | 确认用户名和密码正确。Gmail 必须使用应用专用密码，QQ/163 必须使用授权码 |
| 发送被拒绝 | 邮箱未开启 SMTP 服务 | 登录邮箱 Web 端，在设置中开启 SMTP/POP3 服务 |
| 证书错误 | SSL/TLS 配置不匹配 | 确认端口和加密方式对应正确 |
| 发件人地址被拒绝 | 发件人地址与 SMTP 账户不匹配 | 部分邮件服务器要求发件人地址与登录账户一致 |

### Gmail 应用专用密码设置

1. 登录 Google 账户，前往**安全性**设置。
2. 在"登录 Google"部分，确认已开启**两步验证**。
3. 搜索并进入**应用专用密码**页面。
4. 选择应用类型（邮件）和设备，点击生成。
5. 将生成的 16 位密码填入 SMTP 配置的密码字段。

### QQ / 163 邮箱授权码获取

1. 登录 QQ 邮箱或 163 邮箱的 Web 端。
2. 进入**设置 → 账户**页面。
3. 找到 POP3/IMAP/SMTP 服务，点击**开启**。
4. 按提示进行安全验证（通常需要发送短信）。
5. 获取授权码后，将其填入 SMTP 配置的密码字段（不是邮箱登录密码）。

## 适用场景

- **企业通知邮件**：使用公司域名邮箱（如 `notification@company.com`）发送自动化通知，保持企业品牌一致性。
- **客户沟通邮件**：使用客服邮箱（如 `support@company.com`）发送工单回复和状态更新，客户可直接回复。
- **大批量邮件发送**：通过 Amazon SES 或 SendGrid 等专业邮件服务发送，获得更高的发送配额和更好的送达率。

## 注意事项

- 配置完成后，该空间下**所有**自动化的"发送邮件"操作都会使用自定义 SMTP。如果只想部分工作流使用，目前需要在不同空间中配置。
- SMTP 密码以加密形式存储，其他空间成员无法查看。
- 如果 SMTP 服务器出现故障或凭证过期，邮件发送会失败。建议定期检查并及时更新配置。
- 部分免费邮箱对每日发送量有限制（如 Gmail 每天约 500 封），大批量发送建议使用专业的邮件发送服务。
