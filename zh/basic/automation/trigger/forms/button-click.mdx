<Tip>请注意：所有触发器设置都可以在 AI 对话中完成。告诉 AI 你希望工作流做什么，剩下的交给 AI。</Tip>

用户点击表中按钮字段时触发，工作流获得该行的记录数据。

## 用 AI 创建

打开表格右侧边栏的 AI 对话，描述你的需求。

AI 会为你处理一切：选择合适的触发器，映射相关字段，自动设置所有操作。

只需描述一次目标，工作流即刻就绪，无需手动配置。

**示例：** *"点击审批按钮时更新状态并通知团队"*

## 配置

| 设置 | 必填 | 说明 |
|---|---|---|
| 表格 | 是 | 包含按钮字段的表格 |
| 监听字段 | 是 | 要监控的按钮字段（可选择一个或多个） |
| 筛选条件 | 否 | 仅在记录满足这些条件时触发 |

## 前置条件：创建按钮字段

使用此触发器前，需要先在表格中添加一个按钮类型的字段：

1. 打开目标表格，点击 **+** 添加新字段。
2. 字段类型选择**按钮**（位于"高级"分类下）。
3. 设置按钮的显示文本（如"审批"、"发送"、"同步"）。
4. 保存字段。

按钮字段会在表格的每一行中显示一个可点击的按钮。

## 如何设置

1. 新建工作流，选择**按钮点击时**作为触发器。
2. 选择包含按钮字段的表格。
3. 在监听字段中选择要监控的按钮字段。
4. （可选）添加筛选条件。例如：只在"状态"为"待审批"时响应按钮点击。
5. 添加后续操作步骤。
6. 点击**测试**验证配置。
7. 开启工作流。

## 触发后可用的数据

按钮点击后，后续步骤可以访问该行记录的所有字段数据，包括：

- 记录 ID
- 所有字段的当前值
- 点击时间

你可以在后续步骤中通过 **+** 按钮引用这些数据，比如将记录 ID 传给"更新记录"操作，或将字段值插入邮件正文。

## 适用场景

- **一键审批流程**：在审批表中添加"审批"按钮，点击后自动将状态更新为"已通过"，同时发送通知邮件给申请人，告知审批结果。
- **按需数据同步**：在产品表中添加"同步到商城"按钮，点击后通过 HTTP 请求将该产品的信息推送到外部电商平台的 API。
- **手动触发报告生成**：在项目表中添加"生成报告"按钮，点击后使用 AI 生成操作汇总该项目的关键数据，并将报告内容写入指定字段。
- **客户跟进提醒**：在客户表中添加"安排回访"按钮，点击后自动在任务表中创建一条回访任务，并设置提醒时间。

## 注意事项

- 按钮字段本身不存储数据，它只是一个触发入口。
- 每次点击都会触发一次工作流。如果用户快速连续点击，可能会产生多次运行。可以通过筛选条件（如"状态不等于已处理"）来防止重复处理。
- 按钮点击触发器是唯一由用户主动发起的触发方式，适合需要人工判断和确认的场景。
- 如果表格中有多个按钮字段，每个按钮可以关联不同的工作流。
