# AI语音与Markdown优化

## 功能更新

* AI 文本框新增**实时语音输入**，用户可以直接说话，并实时看到语音转成文字。

* Chat 现在支持选择 **reasoning effort（推理强度）**，可在模型选择器中直接控制回答深度。

* Chat 队列中的引用内容现在以 **标签样式** 展示，不再显示原始标记文本，更容易阅读。

* 编辑队列中的 Chat 消息时，会自动回到输入框，方便继续修改和发送。

* 选区统计结果现在可以直接复制，方便复用计算值。

## 问题修复&优化

* 重新生成 AI 回复时，会使用用户当前选择的 **模型和推理强度**。

* 多选问答卡片新增清晰的 **Next** 按钮，帮助用户更明确地继续流程。

* 修复 v2 中向 **collaborator 字段**添加成员后未触发站内或邮件通知的问题，新分配成员现在能正常收到通知。

* 优化 Markdown 协作编辑体验：其他用户的更新不会打断当前编辑，也不会覆盖正在输入的内容。

* 修复打开并关闭 Markdown 单元格但未修改内容时，可能覆盖远程新内容的问题，降低协作冲突风险。

* 修复在同一记录的多个长文本字段间切换时，展开的 Markdown 编辑器未正确同步当前字段 Markdown 开关状态的问题。

* 修复 Markdown 长文本字段编辑后再次展开时，内容可能短暂消失的问题。

* 修复 Markdown 模式下，整行选中后无法用 Delete 或 Backspace 删除的问题。

* 优化个人设置中的主题卡片加载效果，预览图加载时不再导致卡片布局跳动。

* 优化 Ask User Question 组件：选项内容换行更稳定，在窄布局下也更易阅读。

* 修复 Ask User Question 交互中的动画不一致问题，使切换更稳定自然。

* 修复 Ask User Question 在特定场景下高度异常导致的布局问题。

* 优化 AI Chat 输入流程，减少草稿状态不一致带来的编辑和发送异常。

* 优化了 AI 输入体验，语音输入过程中更稳定，用户可以继续编辑或直接提交内容。

</Update>

# AI Chat 体验与稳定性优化

## 功能更新

* **AI Chat 支持消息排队发送**：在上一条回复生成中也可以继续发送新消息，消息会按顺序发送，交互更连贯。

* **AI Chat 支持大段文本自动转附件**：粘贴较长文本或 Markdown 时，会自动作为文件处理，保持输入框简洁。

* **文本和 Markdown 附件支持预览**：可直接在文件查看器中打开检查内容。

* **新增 Space 数量限制**：单个账号最多拥有 10 个 Space，达到上限后会在产品内显示提示。

## 问题修复&优化

* 提升了账单透明度，更清晰地展示附加组件的额度详情，包括更明确的提示信息和更准确的add-on金额。

* 优化自动化运行记录中的用量统计，使其与账单口径更一致。

* 调整新建 Space 的初始化流程，在 Space 创建完成后再创建默认 AI 集成，提高初始化稳定性。

* AI Chat 术语优化：“Workflows” 改为 “Automations”，“Context” 改为 “Nodes”，更贴近实际使用方式。

* AI Chat 发送状态更清晰：更容易区分消息是等待中、发送中，还是等待当前回复完成后再发送。

* 修复 AI Chat 队列消息在刷新、切换聊天或自动发送时可能短暂消失或丢失的问题。

* 优化 AI 对聊天中表格数据、单元格内容和引用内容的识别，减少 App Builder 无法理解所选数据的情况。

* 修复聊天附件同名文件可能冲突的问题，确保附件标签、AI 处理和下载文件保持一致。

* 恢复管理员侧会话查看器中的附件预览能力，上传文件可再次打开查看。

* 修复 Chat 中已选择的 AI 模型可能未生效的问题，发送消息时会更可靠地使用用户选择的模型。

* 修复卡片式 AI Chat 中手动输入内容未正确提交的问题。

* 优化 AI Chat 上下文标签在深色气泡中的可读性。

</Update>

# AI、自动化与稳定性优化

## 功能更新

* **AI 模型推荐提示**：推荐切换到 GPT-5.5 模型，效果更智能，算力利用更高效。

* **表格选择统计增强**：在网格中选择行、列或大量记录时，统计结果更稳定，统计标签会保持在表格区域内。

* **组织协作者邀请支持按部门搜索**：邀请协作者时可按部门搜索，更快找到目标团队；同名子部门会显示父级路径，便于区分。

* **自动化失败排查更方便**：失败通知邮件可直接跳转到对应运行记录，并自动打开失败详情。

* **自动化失败运行支持手动重跑**：修复配置或动作后，可从运行历史中重新执行失败任务。

* **CLI 创建记录支持更多字段值格式**：创建记录时支持 user、link、attachment 等更多字段值格式，并返回更明确的校验错误。

## 问题修复与优化

* **优化聊天内容展示**：修复聊天序列和列表项中文本过早截断的问题，让对话内容更充分利用可用宽度，阅读更方便。

* **优化公式字段数值比较**：提升数字字段与数字值比较时的稳定性和效率，降低复杂表格中更新缓慢、锁表或超时的风险。

* **提升相关字段类型的比较一致性**：优化 formula、rollup、lookup、rating、auto-number 等字段类型在数值比较时的表现，同时保留文本与数字混合比较的兼容性。

* **修复公式字段异常**：解决 v2 中新建记录后公式字段可能显示为空，或新建公式字段失败的问题，尤其是依赖跨 Base 关联字段的场景。

* **优化跨 Base 引用校验**：减少字段错误误报，避免相关公式结果被错误返回为 `NULL`。

* **修复部分字段误标错误**：已安全恢复受影响 Base 中可确认状态的 35 个字段。

* **提升选择统计准确性**：即使选择范围包含未加载到本地的记录，统计结果也能保持准确。

* **修复分组视图下的选择统计问题**：选择统计会正确遵循分组视图和折叠分组，只统计当前可见记录。

* **修复空值选择统计加载异常**：全为空或全为 `NULL` 的选择不再错误显示为一直加载中。

* **优化 Grid 拖选统计精度**：减少小数和大数字计算中的误差。

* **修复 App Builder 登录问题**：提升 OAuth 登录回调稳定性，减少环境重启后登录失败的情况。

* **优化 App Builder 访问速度**：减少不必要的请求，让应用访问更顺畅。

* **提升 App Builder 的 Agent Computer 创建稳定性**：修复依赖安装问题，减少环境启动失败。

* **修复导入代码后 App 预览未刷新的问题**：导入变更后，预览会立即更新。

* **支持一个登录用户表关联多个 App**：优化相关选择和展示体验。

* **优化深色模式下 App Builder 标签页可读性**：提升对比度和状态区分。

* **修复 HTML 预览标签页高度异常问题**：让预览布局更稳定。

* **优化表格提及体验**：在对话框中提及表格时减少不必要加载，降低界面卡顿。

* **优化字段描述提示**：字段描述 Tooltip 增加最大宽度，长描述不再横向撑满屏幕。

* **修复模板预览权限问题**：共享或模板预览中的 Dashboard 现在可以正常打开，减少意外的 403 错误。
