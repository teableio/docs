Teable 自动化让强大工作流变得简单。无需写代码、无需拖拽、无需复杂配置。你只需用自然语言告诉 AI 你的需求，Teable 会替你处理重复工作。

## 用 AI 构建（从零开始）

最简单的方式。打开任意表格，点击右侧边栏的 **AI 对话**，描述你想要的效果：

- *"新订单进来时，发一条 Slack 通知，包含客户名和金额。"*
- *"每周一早上，给我发一封包含所有逾期任务的邮件。"*
- *"有人提交反馈表单时，自动分类并更新分类字段。"*

AI 会为你创建并测试完整自动化，包括触发器、操作和字段映射。你只需在自动化面板查看结果并启用即可。

## 用 AI 构建（从触发器开始）

你也可以从一个具体的触发器开始，让 AI 来构建操作：

1. **添加触发器** — 选择触发类型（如"记录创建时"）并配置。
2. **添加操作** — 点击 **+**，进入 **用 AI 创建**，选择 **运行脚本**。
3. 在配置面板中，点击 **打开编辑器**。
4. 从 **建议列表** 中快速选择，或用自然语言描述需求。
5. AI 生成 JavaScript 代码，检查后点击 **应用**。
6. 点击 **测试** 用真实数据运行，然后 **启用** 工作流。

## 手动构建

你也可以使用内置操作逐步搭建：

1. **添加触发器** — 选择触发类型，选择表，配置字段。点击 **测试**。
2. **添加操作** — 选择操作类型（创建记录、发送邮件、HTTP 请求等），通过 **+** 插入动态变量。点击 **生成预览** 或 **按配置执行**。
3. **启用** — 点击左上角开关。

## 编辑运行中的工作流

修改会保存为草稿，线上工作流继续使用旧版本，直到你点击 **应用更新**。

## 密钥管理

使用 **密钥管理** 保存自动化运行时需要的 API Key、访问凭证和配置。打开工作流，在编辑标签页点击测试控件旁的 **密钥管理**。

密钥只作用于当前自动化。脚本操作可通过环境变量读取，例如 `process.env.EXTERNAL_API_KEY`。已保存的值不会明文展示；需要修改时，输入新值覆盖即可。

## 运行历史

打开工作流 → **运行历史**。历史面板会列出每次运行的状态、耗时和步骤详情。点击一条运行，可查看每一步的输入和输出。

可按状态、运行耗时或时间范围筛选历史记录。失败运行可在详情中诊断或重跑：

- **诊断** 会把失败运行作为上下文加入 AI 对话，让 Cuppy 帮你分析失败原因。
- **完整重跑** 会重新执行全部步骤，可能产生重复记录、邮件或通知。
- 当 Teable 可以安全地从失败节点继续时，可选择 **从失败步骤恢复**。

当 **从失败步骤恢复** 不可用时，Teable 会说明为什么当前只能使用 **完整重跑**。例如，工作流在本次失败运行后被修改、工作流已停用、失败前没有成功完成的步骤，或缺少控制节点数据。

重跑产生的运行会标记为重试，并可跳转查看原始运行。

## 限制

| 项目 | 限制 |
|---|---|
| 邮件发送速率 | 5 封 / 秒 / Base |
| Webhook 接收速率 | 50 次 / 秒 / Base，2 次 / 秒 / 工作流 |
| Webhook 请求体大小 | 4 MB |
| 运行配额与历史保留 | 按套餐不同 |

  <Accordion title="为什么会收到自动化运行次数提醒？">
    在 Teable Cloud 中，当空间的自动化运行次数接近或超出额度，或短时间内运行次数明显增加时，Teable 会向空间所有者发送用量通知。

    对于配额提醒，Teable 会在当前计费周期的自动化运行次数达到额度的 80% 或 90% 时发送通知。

    专业版或以上计划在自动化运行次数超出额度后，可能会收到提醒。之后可能约每 6 小时提醒一次。

    Teable Cloud 也可能因为异常用量发送关键通知，比如单日运行次数明显增加，或短时间内突然运行很多次。系统会参考近期的日级和小时级运行模式，具体触发阈值可能调整。

    通知会打开**空间设置** → **计费**。如果能识别主要来源，通知会显示哪个工作流和 Base 贡献了最多运行次数。配额提醒基于计入套餐额度的运行次数；异常用量检测也可能参考近期状态为**成功**、**失败**或**等待中**的运行。

    如果这些用量符合预期，可以考虑购买额外自动化额度或调整订阅计划。如果用量不符合预期，请查看主要工作流，并调整或暂停相关自动化。
