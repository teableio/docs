<Tip>Pro 版及以上适用</Tip>

<Note>
自定义模型配置适用于所有 Teable AI 功能，包括 **AI Chat**、**AI 字段**、**App Builder** 和**自动化**。

自 2026 年 4 月 9 日起，Teable 的 Agent 引擎已完成升级，以提升 **AI Chat Agent** 模式和 **App Builder** 的 AI 能力。本次升级后，这些功能当前仅支持 Anthropic 兼容的 API 端点，**使用不兼容的端点可能会导致报错。**

- **Cloud 用户（BYOK）：** 如果您的自定义模型提供商不支持 Anthropic Messages API 格式，AI Chat Agent 和 App Builder 将无法使用您的 BYOK 配置。请切换至 Anthropic 兼容的提供商（如 Anthropic API、OpenRouter），或使用 Teable 默认模型。
- **私有化部署用户：** 请确保您配置的 LLM 端点兼容 Anthropic 格式。您也可以等待我们后续适配 OpenAI 兼容端点后，拉取最新镜像即可。
- **OpenAI 兼容端点支持**已在规划中，将在后续版本中推出。

**推荐使用 Teable Credits：** Teable Credits 当前以成本价提供，没有中间商赚差价，让您以最优惠的价格使用顶级 AI 模型，体验最流畅。
</Note>

配置完成后，添加的模型可以供当前空间内的 **AI 字段**、**自动化**、**AI Chat** 和 **App Builder** 使用。

## 配置入口

1. 打开目标空间。
2. 点击右上角 **设置**。
3. 进入 **AI 配置**。

## 配置步骤

在 **AI 能力** 中按需开启：

- **AI 字段**
- **AI 聊天**

### 添加模型提供商

点击 **添加模型提供商**，填写以下信息：

- **名称**：用于区分不同的模型提供商。
- **提供商类型**：选择服务商类型。
- **基础 URL**：填写服务商的 API 地址。
- **API 密钥**：填写服务商提供的 API 密钥。
- **模型**：填写要接入的模型名称，多个模型之间使用英文逗号分隔。

### 测试模型能力

当前支持三种测试方式：

- 在模型提供商行点击 **测试**。
- 在单个模型行点击 **测试**。
- 在列表右上角点击 **测试模型能力**，批量测试全部已配置模型。

如果某个模型用于图片生成，请先勾选 **生图模型** 再执行测试。勾选后会按生图模型测试 **文生图** 和 **图生图** 能力；未勾选时，则按普通文本模型测试。

## 填写提示

### 基础 URL

<Warning>请填写 API 端点地址，而不是服务商官网地址。多数 OpenAI 兼容接口需要以 `/v1` 结尾；除非服务商文档明确要求，否则不要在 URL 末尾额外加 `/`。</Warning>

<Warning>Teable 不支持 coding plan 的 Key。请使用提供商后台创建的标准 API Key。因为 coding plan 的 Key 通常只能在指定的 coding 工具中使用，不能作为通用 API Key 接入 Teable。若您使用这类 Key，模型测试失败是正常现象。</Warning>

常见基础 URL 示例：

| 提供商 | 基础 URL 格式 |
|--------|--------------|
| Anthropic | `https://api.anthropic.com/v1` |
| OpenAI | `https://api.openai.com/v1` |
| Google | `https://generativelanguage.googleapis.com/v1beta` |
| DeepSeek | `https://api.deepseek.ai/v1` |
| Azure | `https://{您的资源名称}.openai.azure.com` |
| Mistral | `https://api.mistral.ai/v1` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| 智谱 AI | `https://open.bigmodel.cn/api/paas/v4` |
| XAI (Grok) | `https://api.x.ai/v1` |
| OpenRouter | `https://openrouter.ai/api/v1` |
| TogetherAI | `https://api.together.xyz/v1` |
| Ollama（本地） | `http://localhost:11434` |

### 模型

<Note>模型名称必须与服务商文档一致，并区分大小写。多个模型之间使用英文逗号分隔。</Note>

常见模型名称示例：

| 提供商 | 模型名称示例 |
|--------|--------------|
| OpenAI | `gpt-5.4, gpt-5.4-mini, gpt-5` |
| Anthropic | `claude-opus-4-6, claude-sonnet-4-6, claude-haiku-4-5` |
| Google | `gemini-3.1-pro-preview, gemini-3-flash, gemini-2.5-flash` |
| Azure | `gpt-5.4, gpt-5, gpt-5-mini, gpt-4o` |
| DeepSeek | `deepseek-chat, deepseek-reasoner` |
| XAI | `grok-4, grok-4.1-fast` |
| 通义千问 | `qwen3.5-plus, qwen3-max` |
| OpenRouter | `anthropic/claude-opus-4-6, google/gemini-3.1-pro-preview` |
| TogetherAI | `deepseek-ai/DeepSeek-R1, meta-llama/Llama-4-Maverick-17B-128E-Instruct-FP8` |
| Mistral | `mistral-large-latest, mistral-medium-latest, codestral-latest` |
| Ollama | `qwen3.5:9b, gemma3:12b, llama3.2:8b` |

注意事项：

- 模型名称区分大小写，请使用服务商文档中的准确名称。
- 部分服务商要求逗号后不要加空格，例如 `gpt-4o,gpt-4-turbo`。
- OpenRouter 格式为 `提供商/模型名称`，例如 `openai/gpt-4o`。
- Azure 请使用您在 Azure AI Foundry 或 Azure OpenAI Studio 中创建的部署名称，而不是基础模型名。

## 常见问题

<AccordionGroup>
<Accordion title="所有模型测试失败">
先检查 **基础 URL** 是否正确，是否误填成官网地址，或在末尾多加了 `/`。如果使用的是 OpenAI 兼容接口，也请确认地址是否缺少 `/v1`。
</Accordion>

<Accordion title="出现“测试失败”错误">
检查 **API 密钥** 是否有效，以及账户是否仍有额度或调用权限。
</Accordion>

<Accordion title="连接超时">
检查 **基础 URL** 是否正确，且当前环境可以访问这个地址。
</Accordion>

<Accordion title="找不到模型">
确认 **模型** 与服务商文档完全一致，包括大小写和分隔格式。
</Accordion>

<Accordion title="使用 coding plan 的 Key 测试失败">
Teable 不支持这类 Key。请到服务商后台重新创建标准 **API 密钥** 后再测试。
</Accordion>

<Accordion title="生图模型测试结果不对">
如果这个模型本来用于图片生成，请先勾选 **生图模型**，再重新测试。勾选后会改为测试文生图和图生图能力。
</Accordion>
</AccordionGroup>
