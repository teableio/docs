title: "跨 Base 自动同步记录"
description: "使用跨 Base 访问自动将新记录或更新的记录复制到另一个 Base"
---

<Tip>我们推荐直接在 AI 对话中构建自动化示例。只需描述你想要的工作流，AI 会为你完成全部设置，包括触发器、操作、脚本和配置。</Tip>

## 用 AI 创建

打开表格右侧边栏的 AI 对话，告诉 AI 你想要什么。例如：

*"销售 Base 中新建订单时，在履约 Base 中创建对应记录，包含订单号、客户名、金额，状态设为待履约"*

AI 会自动创建完整的工作流。你可以查看生成的脚本，用真实数据测试，确认无误后启用。

## 手动设置

1. 在**销售** Base 中**创建工作流**，触发器选择 **记录创建时**。
2. 选择你的**订单**表。
3. 点击**测试**。
4. **添加操作** → **创建记录**。
5. 点击表格选择器旁的**跨 Base 访问**。
6. 切换到**履约** Base，选择**订单**表。
7. 映射字段：
   - 订单号 → 触发器的订单号（通过 **+** 插入）
   - 客户 → 触发器的客户
   - 金额 → 触发器的金额
   - 状态 → `待履约`
8. 点击**测试**，然后**启用**。

## 双向同步

要实现双向更新同步：
1. 在**履约** Base 中创建第二个工作流。
2. 使用**记录更新时** → **更新记录**，通过跨 Base 访问指向销售 Base。
3. 使用筛选条件避免无限循环——例如仅在特定字段变更时触发，并在更新前检查值是否确实不同。
