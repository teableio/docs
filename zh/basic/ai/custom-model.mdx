<Tip>Pro 版及以上适用</Tip>

## 配置入口

1. 打开目标空间。
2. 点击右上角 **设置**。
3. 进入 **AI 配置**。

<img
  src="/images/docs/ai/2026-05-21-custom-model-ai-settings.png"
  alt="AI 配置页面"
  className="docs-screenshot"
/>

## 配置步骤

在 **AI 能力** 中按需开启：

- **AI 字段**
- **AI 聊天**

开启 **AI 聊天** 表示允许空间使用聊天功能。

### 添加模型提供商

<Note>
添加新的 LLM 提供商时，提供商类型请选择 **OpenAI**、**Anthropic** 或 **OpenAI Compatible**（OpenAI 兼容）。如果服务商提供 OpenAI 兼容接口，请选择 **OpenAI Compatible**。如果任务涉及图片、截图等视觉输入，请选择支持视觉能力的模型；不支持视觉的模型会导致这类多模态任务报错。
</Note>

点击 **添加模型提供商**，填写以下信息：

- **名称**：用于区分不同的模型提供商。
- **提供商类型**：选择服务商类型。
- **基础 URL**：填写服务商的 API 地址。
- **API 密钥**：填写服务商提供的 API 密钥。
- **模型**：填写要接入的模型名称，多个模型之间使用英文逗号分隔。

  src="/images/docs/ai/2026-05-21-custom-model-add-provider.png"

### 测试模型能力

当前支持三种测试方式：

- 在模型提供商行点击 **测试**。
- 在单个模型行点击 **测试**。
- 在列表右上角点击 **测试模型能力**，批量测试全部已配置模型。

如果某个模型用于图片生成，请先勾选 **生图模型** 再执行测试。勾选后会按生图模型测试 **文生图** 和 **图生图** 能力；未勾选时，则按普通文本模型测试。

## 填写提示

### 基础 URL 和模型

<Warning>部分 Coding Plan Key 可能只允许在指定开发工具中使用，并不是标准 API Key。是否可用于第三方服务，请以对应提供商的条款和说明为准；如需接入 Teable，请使用提供商后台创建的标准 API Key。</Warning>

常见填写示例：

| 提供商类型 | 基础 URL 格式 | 模型名称示例 |
|--------|--------------|--------------|
| OpenAI | `https://api.openai.com/v1` | `gpt-5.5,o3,gpt-5-mini` |
| Anthropic | `https://api.anthropic.com/v1` | `claude-opus-4-8,claude-sonnet-5,claude-haiku-4-5` |
| OpenAI Compatible | 对应服务商的 OpenAI 兼容 `/v1` 接口 | `gpt-5.5,gpt-5.4,o3,gpt-5-mini` |

<Note>模型名称必须与服务商文档一致，并区分大小写。多个模型之间使用英文逗号分隔。</Note>

注意事项：

- 模型名称区分大小写，请使用服务商文档中的准确名称。
- 部分服务商要求逗号后不要加空格，例如 `gpt-5.5,o3`。
- 使用 **OpenAI Compatible** 提供商时，请按对应服务商要求填写模型名称。有些服务会使用 `provider/model-name` 格式。

## 常见问题

<AccordionGroup>
<Accordion title="所有模型测试失败">
先检查 **基础 URL** 是否正确，是否误填成官网地址，或在末尾多加了 `/`。如果使用的是 OpenAI 兼容接口，也请确认地址是否缺少 `/v1`。
</Accordion>

<Accordion title="出现“测试失败”错误">
检查 **API 密钥** 是否有效，以及账户是否仍有额度或调用权限。
</Accordion>

<Accordion title="连接超时">
检查 **基础 URL** 是否正确，且当前环境可以访问这个地址。
</Accordion>

<Accordion title="找不到模型">
确认 **模型** 与服务商文档完全一致，包括大小写和分隔格式。
</Accordion>

<Accordion title="使用 coding plan 的 Key 测试失败">
这类 Key 可能不支持标准 API 调用，或提供商条款不允许用于第三方服务。请先查看对应 Coding Plan 的条款；如需接入 Teable，请使用服务商后台创建的标准 **API 密钥**。
</Accordion>

<Accordion title="生图模型测试结果不对">
如果这个模型本来用于图片生成，请先勾选 **生图模型**，再重新测试。勾选后会改为测试文生图和图生图能力。
</Accordion>
</AccordionGroup>
