## 1. 配置 LLM API

选择 AI 模型的接入方式，支持以下两种选项：

### AI Gateway（推荐）

通过 Vercel AI Gateway 接入数百个模型

- **AI Gateway API Key**（必填）：从 Vercel AI Gateway 控制台获取。
- **自定义 Gateway URL**（可选）

### 自定义提供商

接入自托管或私有模型，支持多个提供商（OpenAI、Anthropic、Azure 等）。

- **+ 添加 LLM 提供商** 填写提供商信息、API Key 及模型列表。
- 点击 **测试** 按钮可验证连通性。
- 每个模型会显示其支持的能力：**图片**、**PDF**、**网络搜索**、**工具调用**、**推理**、**图片生成**。
- 对于生成图片而非文本的模型，勾选 **图片生成模型**。

配置完成后，点击 **测试模型能力** 按钮，系统会自动检测每个模型支持的功能，确保图片和 PDF 处理等能力被正确识别。

完成后点击 **保存并继续**。

---

## 2. 配置推荐模型

设置用户在 AI 字段和自动化中可选择的模型。使用 AI Gateway 时需手动选择模型；使用自定义提供商时，模型在第 1 步中管理。

---

## 3. 设置对话模型

手动选择侧边栏 AI 对话功能使用的模型。

- **快捷设置**：点击 **使用推荐** 按钮，一键将推荐模型设为所有对话场景的默认模型。
- **对话模型**：通过下拉菜单从推荐模型池中手动选择特定模型。

## 应用构建器

配置相关设置以启用 [应用构建器](/zh/basic/ai/app-builder) 功能，从您的数据生成自定义 Web 应用程序，由 v0 驱动。

- **v0**：Vercel 的 AI 应用生成平台，负责生成应用 UI 和组件。
- **Vercel**：应用部署平台，发布时 Teable 通过 Vercel 将应用部署到生产环境。

<Warning>
请确保 Teable 可被公开访问，因为 v0 和 Vercel 需要访问本实例的 Teable API。可使用 **测试公开访问** 按钮进行验证。
</Warning>

### 必需配置

| 配置项 | 说明 |
|--------|------|
| **v0 API 密钥** | 从 [v0.app/chat/settings/keys](https://v0.app/chat/settings/keys) 获取 |
| **Vercel API 密钥** | 从 [vercel.com/account/settings/tokens](https://vercel.com/account/settings/tokens) 获取 |

<Warning>
v0 API 需要**付费订阅**才能使用。免费额度不足以支持 API 调用。如果创建应用时出现「Internal Server Error」错误，请升级您的 v0 订阅计划。
</Warning>

### 自定义域名（可选）

应用构建器构建的应用部署在 Vercel 上。有两种方式使用自定义域名：

#### 方式 A：Vercel 域名（自动配置）

从 Vercel 购买的域名会自动完成绑定配置。

1. 从 [Vercel Domains](https://vercel.com/domains) 购买域名
2. 在管理员后台的 **自定义域名** 字段中输入域名

#### 方式 B：外部域名（手动 DNS 配置）

如果您已经从其他注册商（如 Cloudflare、GoDaddy）购买了域名：

1. 在 Teable 管理员后台，在 **自定义域名** 中输入您的根域名（如 `example.com`）
2. 添加您的 **Vercel API Token**
3. 发布应用时，从 URL 中记录 **App ID**（如 `appXXX123`）
4. 在您的 DNS 服务商中为每个应用添加 CNAME 记录：

| 字段 | 值 |
|------|-----|
| 类型 | CNAME |
| 名称 | App ID（如 `appXXX123`） |
| 目标 | `cname.vercel-dns.com` |
| 代理 | 仅 DNS（如使用 Cloudflare） |

5. 在 Vercel 控制台（项目 → 设置 → 域名）中添加完整域名（如 `appXXX123.example.com`）并点击 **Verify**
6. 验证通过后，HTTPS 将自动配置

<Note>
每个应用会获得一个基于 App ID 的唯一子域名。不支持路径形式的 URL（如 `yourdomain.com/app/xxx`）。
</Note>

### API 代理（可选）

对于访问 v0/Vercel API 受限的地区，可以配置代理 URL：

| 配置项 | 说明 |
|--------|------|
| **v0 Base URL** | v0 API 的代理地址 |
| **Vercel Base URL** | Vercel API 的代理地址 |

### AI Gateway（可选）

如果您使用 [Vercel AI Gateway](https://vercel.com/docs/ai-gateway)，可以在这里配置 API 密钥以启用：

- AI 请求的集中监控和日志记录
- 速率限制和成本控制
- 缓存以降低 API 成本

| 配置项 | 说明 |
|--------|------|
| **AI Gateway API Key** | 从 Vercel AI Gateway 设置中获取 |
