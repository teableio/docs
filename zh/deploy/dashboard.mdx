---
title: "开启 Standalone Dashboard"
description: "自部署用户如何开启 Standalone Dashboard 功能"
---

<Callout type="warning">
Standalone Dashboard 功能已被 App Builder 取代。我们强烈推荐你使用 App Builder 来构建更强大、可自定义的仪表盘，以获得最佳体验。但如果你无法接入 v0 服务来搭建仪表盘，你可以通过以下配置步骤来启用 Standalone Dashboard 作为替代方案。
</Callout>

## 概述

对于自部署的 Teable 实例，Standalone Dashboard 功能默认是禁用的。如果你需要开启，请按照以下步骤操作。

## 操作步骤

### 第一步：执行 SQL 语句

连接到 Teable 使用的 PostgreSQL 数据库，执行以下 SQL 语句将 `disallowDashboard` 设置为 `false`：

```sql
UPDATE setting SET content = 'false' WHERE name = 'disallowDashboard';
```

### 第二步：清理 setting 表缓存

执行完 SQL 后，需要清理 `setting` 表的缓存才能使配置生效。

由于系统没有直接清理缓存的入口，你可以通过**任意修改一个实例设置选项**来触发缓存刷新。例如：

1. 进入**管理员后台** > **实例设置**
2. **开启**邮箱验证选项
3. 再**关闭**邮箱验证选项

<Callout type="info">
这一步的目的是触发 setting 表的缓存刷新，任意切换一个设置选项即可，不需要真正修改某个功能配置。
</Callout>

完成以上两步后，刷新页面即可看到 Standalone Dashboard 功能已启用。
