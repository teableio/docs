每个数据库可以包含多张表格。表格是一种结构化数据存储方式，用于存储数据库内相关数据的集合。

每个表格通常会包含多个行和列，其中每一列代表一个字段及其属性，而每一行代表一个记录或数据项。表格是数据库的核心组成部分，用于收集、存储和处理数据以便后续的检索、操作和构建数据库。

Teable的表格等同于数据库中表的概念，这个概念有助于理解[Teable和传统电子表格之间的差异](/zh/compare/teable-vs-excel)，以便更好地掌握Teable。

2. 点击目录中的+号按钮

2. 点击菜单中的删除按钮

### 回收站

点击数据库左上方回收站按钮，即可进入表格回收站界面并执行以下操作。

> 回收站中的表格数据仍可被数据库连接所访问

* 清空回收站：永久删除所有的表格数据
* 恢复：将表格恢复到数据库中去

![](/images/ji-ben-gong-neng/d95c9e75be841b6e.png)

### 设计

点击菜单中的设计按钮，进入表格设计页面，可查看表格的基础属性，包括：

**表格相关**

* 表格所属数据库 Schema
* 表格名称，以及在物理表中的名称
* 表格描述
* 修改时间

**字段信息**

字段详细属性

* id 字段 ID
* name 字段名称
* dbFieldName 字段在物理数据库中的名称
* type 字段类型
* description 字段描述
* graph 查看当前字段依赖关系
* cellValueType 当前字段值类型
* isLookup 是否为从关联表查找出来的字段
* isMultipleCellValue 是否为[数组值字段](/zh/basic/field/common/is-multiple-value)
* isComputed 是否为计算字段（计算字段的记录值不可修改）
* isPending 是否为计算中
* hasError 是否存在计算错误
* notNull 是否启用非空校验
* unique 是否启用唯一值校验

### 访问表格的方式

用户可通过三种方式访问表格：

* [视图](/zh/basic/view)
* [API](/zh/api-doc)
* [SQL查询语句](/zh/api-doc/sql-query)

视图是最直观的访问方式。用户可通过各种视图对表格进行操作，并构建复杂的应用。

新添加的表格默认包含[表格视图](/zh/basic/view/grid)，便于用户快速开始处理数据。

### 表格的功能

**创建视图**：用户可创建多种视图操作表格，满足不同场景需求。例如：表格视图数据密度高，操作接近电子表格，易于上手；看板视图为项目管理提供更好支持。
