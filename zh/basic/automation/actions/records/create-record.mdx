---
title: "创建记录"
description: "在工作流中向任意表格创建新记录"
---

<Tip>
我们强烈建议使用运行脚本来搭建自动化，因为它可以覆盖所有操作行为，包括原本需要手动搭建的操作。你只需在 AI 对话中描述需求即可。

请注意：如果你手动添加操作，AI 后续将无法识别或修改它们。
</Tip>

## 用 AI 创建

打开表格右侧边栏的 AI 对话，描述你的需求。

AI 会为你处理一切：选择合适的触发器和操作，映射字段，自动配置整个工作流。

只需描述一次目标，工作流即刻就绪，无需手动配置。

在指定表格中创建一条新记录。你可以将前序步骤的数据映射到新记录的字段中，实现数据的自动流转。创建成功后，新记录的 ID 和字段值可供后续步骤引用。

## 配置

| 设置 | 必填 | 说明 |
|---|---|---|
| 表格 | 是 | 目标表格——即新记录要创建在哪个表中 |
| 字段映射 | 是 | 将每个字段映射为静态值或前序步骤的变量 |

## 如何设置

1. 在工作流中添加**创建记录**操作。
2. 选择目标表格。
3. 系统会列出目标表格的所有字段。为需要填写的字段设置值：
   - **静态值**：直接输入固定文本、数字等。
   - **变量**：点击 **+** 按钮，从触发器或前序步骤中选择数据。例如，将触发记录的"客户名称"字段映射到新记录的"客户名称"字段。
4. 确保所有必填字段都已映射。
5. 点击**测试**验证记录是否能正确创建。

## 字段映射详解

字段映射是将前序步骤的数据"传递"到新记录对应字段的过程。支持的映射方式：

| 字段类型 | 映射说明 |
|---|---|
| 文本 | 直接映射文本值，支持拼接多个变量 |
| 数字 | 映射数字值，确保传入的是有效数字 |
| 单选 / 多选 | 映射选项的文本值，值必须与目标表格中已有的选项匹配 |
| 日期 | 映射日期字符串（如 `2025-03-15`）或日期变量 |
| 关联 | 映射关联记录的 ID |
| 附件 | 映射附件 URL |

<Tip>你可以在一个文本字段中组合多个变量。例如：在"摘要"字段中插入 `触发记录的客户名 - 触发记录的订单号`。</Tip>

## 必填字段说明

<Warning>如果目标表格中存在必填字段但未在映射中赋值，操作会执行失败。</Warning>

建议在配置前先确认目标表格的字段设置，确保所有必填字段都有对应的值来源。如果某个必填字段的值需要根据条件变化，可以在前序步骤中先计算好，再映射到创建记录操作。

## 输出数据

创建成功后，该步骤会输出：

- **记录 ID**：新创建记录的唯一标识，可用于后续的"更新记录"操作。
- **所有字段值**：新记录中每个字段的值。

后续步骤可以通过 **+** 按钮引用这些输出数据。例如，创建记录后立即更新关联表中的某个字段，指向这条新记录。

## 适用场景

- **表单数据转存**：表单提交后，将数据自动写入另一个表格（如归档表或统计表），保持原表结构不变的同时实现数据分发。
- **自动创建关联任务**：在项目管理中，当新项目创建时，自动在任务表中创建多条初始任务（需求分析、设计评审、开发排期等），配合循环执行实现批量创建。
- **跨表数据复制**：当一个表中的记录满足特定条件时，自动在另一个表（甚至另一个 Base）中创建对应记录，实现跨团队的数据共享。
- **邮件内容归档**：收到邮件后，自动将发件人、主题、正文等信息创建为一条记录，方便团队集中管理和后续跟进。

## 注意事项

- 支持[跨 Base 访问](cross-base)，可以在其他 Base 的表格中创建记录。
- 支持[循环执行](../logic/loop-run)，可以批量创建多条记录。
- 通过该操作创建的记录会触发同一表格上的"记录创建时"触发器。如果两个工作流互相创建记录，可能导致无限循环，请务必注意。
- 如果字段映射中引用了前序步骤的变量，但该变量在运行时为空值，则对应字段会留空（非必填字段不会报错）。
- 未映射的字段将使用表格的默认值（如果有），否则留空。
- 关联字段需要传入记录 ID，而不是显示值。如需关联某条记录，请确保传入的是记录 ID 而非记录名称。
