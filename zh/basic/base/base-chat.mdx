## 第一部分：提问的艺术——将业务问题转化为 AI 指令

与 AI 对话的核心，在于将您脑海中的业务问题，转化为 AI 能理解并执行的清晰指令。这不仅是提问，更是一种战略性的信息挖掘。

### 策略一：从“数据盘点”到“日常体检”

不要只在需要时才想起 AI。把它当作您业务的“日常体检”工具，快速掌握核心指标。

<Tabs>
  <Tab title="传统方式">
    打开多个表格，手动计算关键指标，再复制粘贴到报告或群聊中，耗时且容易出错。
  </Tab>
  <Tab title="AI 实践">
    <CodeGroup>
      ```bash title="每日站会/周会前提问"
      1. 给我看看昨天的新增客户数
      ---
      2. 计算一下上周的总销售额和平均客单价
      ---
      3. 哪个销售是上个季度的销售冠军？
      ```
    </CodeGroup>
  </Tab>
</Tabs>

<Tip>
  **经验分享**：将这些问题变成您的工作习惯。每天早上花两分钟，向 AI 询问关键指标，就像看天气预报一样，让您对业务的健康状况了如指掌。
</Tip>

### 策略二：从“大海捞针”到“精准打击”

筛选的价值在于聚焦。当您有了特定的目标或假设，用精准的条件来验证它。

**场景：策划一场针对高价值客户的营销活动。**

1.  **定义“高价值”**:
    `“查询一下，过去一年里累计消费金额超过10000元的客户有哪些？”`
2.  **增加地域限制**:
    `“在这些人里，筛选出华东地区的客户”`
3.  **获取行动信息**:
    `“把他们的姓名和联系电话列出来”`

<Note>
  **最佳实践**：像剥洋葱一样层层深入。先用一个宽泛的条件圈定范围，再逐步增加限定词，最终锁定最精准的目标群体。这比一开始就设定一个超复杂的长问题要高效得多。
</Note>

### 策略三：从“孤立数据”到“发现故事”

任何单一的数据点都是孤立的，但当您将它们连接起来，就能发现趋势、规律和故事。

**场景：季度复盘会议，评估增长策略是否有效。**

<CodeGroup>
  ```bash title="季度复盘分析"
  # 评估过去
  比较一下第一季度和第二季度的利润差异，用具体数字和百分比展示
  ---
  # 洞察趋势
  生成一张折线图，分析过去6个月销售额的增长趋势
  ---
  # 对比分析
  看看‘新品A’和‘经典款B’在过去三个月的销量走势对比
  ```
</CodeGroup>

<Info>
  **战略思维**：**比较**是为了评估过去（策略是否有效），**趋势**是为了预测未来（增长能否持续）。当您向 AI 提问时，心中要带着“So What?”（这说明了什么？）的思考。例如，看到增长趋势放缓，您的下一个问题就应该是：`“分析一下增长放缓的月份，主要受哪个产品线销量下滑的影响？”`
</Info>

### 提问技巧的终极总结：像与真人分析师沟通一样

1.  **明确你的目标**：
    不要问 `“销售情况怎么样？”`，而是问 `“上个月华南区的销售额是多少？”`。

2.  **使用表格内的语言**：
    如果你的列名是“订单金额”，就用“订单金额”而不是“销售额”，这样 AI 的理解会更精确。

3.  **一次只问一个核心问题**：
    这便于您进行追问和深入挖掘。

4.  **善用追问**：
    AI 拥有对话记忆。得到初步答案后，用 `“那...”`，`“其中...”`，`“按...细分”` 来深挖。

## 第二部分：可视化表达——让数据自己开口说话

图表是浓缩的信息，是最高效的沟通语言。您无需学习复杂的图表工具，只需告诉 AI 您的“表达意图”。

<Tip>
  **核心策略：为你的“观众”选择合适的图表**
  
  在提问前，先想一想：**我想通过这张图告诉别人什么？**
</Tip>

<CardGroup>
    <Card title="柱状图：用于比较" icon="chart-column">
    当您想直观比较不同类别的数据大小时使用。
    
    **场景**：向销售团队展示业绩。
    
    **指令**：`“用柱状图展示每个销售上个季度的业绩排名”`
    </Card>
    <Card title="折线图：用于看趋势" icon="chart-line">
    当您想观察数据随时间的变化趋势时使用。
    
    **场景**：向管理层汇报业务健康度。
    
    **指令**：`“生成过去一年我们用户增长数的折线图”`
    </Card>
    <Card title="饼图：用于看构成" icon="chart-pie">
    当您想展示整体中各个部分的占比时使用。
    
    **场景**：分析市场来源或预算分配。
    
    **指令**：`“用饼图显示我们客户的来源渠道占比”`
    </Card>
    <Card title="热力图：洞察密度与强度" icon="fire">
    当您想发现二维数据中的高密度区域和模式时使用。
    
    **场景**：分析用户活跃时间，以便在最佳时段推送内容。
    
    **指令**：`“用热力图展示过去一个月，用户在一周内不同时段的活跃度分布”`
    </Card>
    <Card title="雷达图：进行多维比较" icon="radar">
    当您想从多个维度综合评估和比较一个或多个对象时使用。
    
    **场景**：对比几款竞品或自家产品的综合性能。
    
    **指令**：`“用雷达图比较‘旗舰款X’和‘青春版Y’在性能、续航、拍照、设计和价格五个维度的得分”`
    </Card>
    <Card title="散点图：探索变量关系" icon="braille">
    当您想探究两个数值变量之间是否存在关联性时使用。
    
    **场景**：验证市场投入与销售额之间是否存在正相关关系。
    
    **指令**：`“用散点图分析我们过去12个月的广告投入和对应销售额之间的关系”`
    </Card>
</CardGroup>

<Warning>
- **饼图使用提醒**：当分类超过5个时，饼图会变得混乱。此时可以尝试用柱状图，或者让 AI `“用饼图展示前五大来源渠道的占比，其余的归为‘其他’”`。
- **雷达图**：维度不宜过多（通常3-8个为佳），否则图形会难以解读。
</Warning>


### 最佳实践：从“提问”到“报告”的无缝衔接

![AI对话功能生成图表动图](/images/2025-06-16-22-20.gif)

想象一下，您正在准备一份周报，通过以下对话流，一份有理有据、图文并茂的报告就基本成型了：

1.  **获取核心数据**: `“上周的总收入和新用户数分别是多少？”`
2.  **维度分解**: `“按产品类别细分一下收入构成”`
3.  **生成图表**: `“用柱状图展示各个产品类别的收入”`
4.  **洞察根源**: `“C类产品收入为什么这么低？看看它的退货率是不是最高的”`
