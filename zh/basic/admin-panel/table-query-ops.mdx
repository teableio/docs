<Tip>私有化部署商业版及以上适用</Tip>

路径：管理面板 → 表格查询运维

**表格查询运维** 帮助实例管理员检查可能影响大表性能的查询模式。它会分析保存视图中的筛选、排序、分组，以及关联字段查询配置，并输出查询风险和索引建议。

这个页面用于诊断，不会自动执行索引变更。

## 分析范围

填写下面任意一个 ID，然后点击 **分析**：

| 字段 | 适用场景 |
| --- | --- |
| **空间 ID** | 扫描一个空间内的查询结构。 |
| **Base ID** | 只分析一个数据库。 |
| **表 ID** | 已经知道问题表时，检查单张表。 |
| **最大数量** | 限制返回的候选索引数量。 |

未填写范围 ID 时，**分析** 按钮不可用。

## 查看结果

分析完成后，页面会显示：

| 区域 | 含义 |
| --- | --- |
| 摘要卡片 | 当前范围内扫描到的保存视图、关系字段、候选索引、已验证建议和未推荐候选数量。 |
| 覆盖报告 | 显示已解析、已校验、已跳过和未推荐的来源数量。 |
| 查询风险 | 显示查询结构、风险等级、原因、执行计划证据和后续动作。 |
| 最终推荐索引 | 已通过验证、可供管理员继续确认的候选索引。 |
| 未推荐候选 | 被分析过但暂不推荐的候选索引。 |

<Info>分析结果显示查询配置、字段/索引结构和执行计划摘要，不展示记录内容或单元格值。</Info>

## 注意事项

- 排查单个慢数据库或慢表时，先使用较小的范围。
- 索引建议仍需要管理员确认后，才能作为数据库变更执行。
- 结果取决于当前保存视图和关联字段配置。
