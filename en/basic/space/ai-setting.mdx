### 1. AI Capability Settings

- Description:
  - Space administrators can independently control whether each AI feature is available within the space.
  - Features in the list include:
    - AI Basic Capability: AI basic capability for table queries, SQL execution, script generation, and data visualization
    - Create Base: Create Base from scratch
    - Create App: Create App based on Base
    - Create Automation: Create Automation from scratch
    - Base Resource Operation: Create, modify, and delete tables, views, fields, and records through ai
    - Suggestions: Give suggestions based on the current database data and conversation content

![](/images/2025-09-29-23-13-31.png)

### 2. Custom Model

- Used to activate or deactivate all custom AI API configurations within the current space. When enabled, space AI configurations will take priority.
- Description:
  - Off: Use official default AI models.
  - On: Use custom configured AI models (consumes your own tokens). Please configure the Advanced Chat Model before enabling.


> [Configure Custom Model Guide](/en/basic/ai/custom-model)

### 3. LLM Providers

- Manage the integration of third-party AI model providers.



![](/images/2025-10-10-19-44-41.png)

### 4. Model Preferences

- Specify default chat models for different types of task scenarios to optimize cost and performance.
  - Advanced Chat Model:
    - Used for AI Chat primary conversation, complex reasoning, and code generation tasks.
    - Recommended: `claude-sonnet-4.5` or models with strong reasoning capabilities.
  - Medium Chat Model:
    - Used for AI formula generation.
    - Falls back to Advanced Chat Model if not configured.
    - Recommended: `claude-haiku-4.5`
  - Small Chat Model:
    - Used for lightweight tasks such as auto-generating chat titles and base names.
    - Falls back to Advanced Chat Model if not configured.
    - Recommended: `claude-haiku-4.5`

### 5. Advanced Chat Model Capabilities

- Test the multimodal processing capabilities of the selected "Advanced Chat Model".
- Description:
  - You can use the "Test Model" button to verify whether the selected advanced model supports processing images and PDF content.

### 6. Embedding Model

- Optional. Used for document Q&A - when users upload documents and ask questions about their content. Usually, the model ID contains "embedding".

![](/images/2025-09-29-23-13-17.png)

### 7. App Builder

Configure the v0 API Key to enable [App Builder](/en/basic/ai/app-builder) capabilities. Access [v0.app/chat/settings/keys](https://v0.app/chat/settings/keys) to obtain your API Key.

### 8. Web Search
- Description: Configure Firecrawl API key to enable web search feature, access Firecrawl settings to get API key
