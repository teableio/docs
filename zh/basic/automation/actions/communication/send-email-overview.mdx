<Tip>
我们强烈建议使用运行脚本来搭建自动化，因为它可以覆盖所有操作行为，包括原本需要手动搭建的操作。你只需在 AI 对话中描述需求即可。

请注意：如果你手动添加操作，AI 后续将无法识别或修改它们。
</Tip>

## 用 AI 创建

打开表格右侧边栏的 AI 对话，描述你的需求。

AI 会为你处理一切：选择合适的触发器和操作，映射字段，自动配置整个工作流。

只需描述一次目标，工作流即刻就绪，无需手动配置。

此操作向一个或多个收件人发送邮件。你可以自定义邮件的每个部分 — 收件人、主题、正文、附件 — 并使用前序步骤中的动态变量来个性化每条消息。

默认使用 Teable 内置邮件服务发送。如需从自己的域名发送，请配置[自定义 SMTP 服务器](smtp-sender)。

## 配置

| 设置 | 必填 | 说明 |
|---|---|---|
| 收件人（To） | 是* | 收件人邮箱地址，多个地址用逗号分隔 |
| 主题（Subject） | 是 | 邮件主题行，支持通过 **+** 插入动态变量 |
| 正文（Body） | 是 | 邮件正文，支持 Markdown、HTML 和动态变量 |
| 发件人姓名（Sender Name） | 否 | 收件人收件箱中显示的名称（如"Acme 客服"） |
| 抄送（CC） | 否 | 抄送收件人 |
| 密送（BCC） | 否 | 密送收件人 |
| 回复地址（Reply-To） | 否 | 收件人回复时使用的地址（如果与发件人不同） |
| 附件（Attachments） | 否 | 来自前序步骤或附件字段的文件附件 |

\* **收件人**和**密送**至少填写一项。

**速率限制：** 每个 Base 每秒 5 封邮件。

## 如何设置

1. 在工作流中添加**发送邮件**操作。
2. 在**收件人**字段中，输入静态邮箱地址或点击 **+** 插入动态值（例如触发记录中的邮箱字段）。
3. 编写**主题**。点击 **+** 插入动态值，如记录名称或状态。示例：`来自 {{客户名称}} 的新订单 — #{{订单号}}`。
4. 编写**正文**。你可以使用：
   - **纯文本** — 直接输入消息内容。
   - **Markdown** — 使用 `**粗体**`、`*斜体*`、`- 列表`、`[链接](url)` 等语法。
   - **HTML** — 直接编写 HTML 标签，完全控制格式。
   - **动态变量** — 在正文任意位置点击 **+** 插入前序步骤中的值。
5. （可选）根据需要设置**发件人姓名**、**抄送**、**密送**或**回复地址**。
6. （可选）通过引用前序步骤中的附件字段来添加**附件**。
7. 保存操作。

## 在主题和正文中使用变量

变量可以让每封邮件实现个性化。点击任意文本字段中的 **+** 按钮打开变量选择器。你可以引用：

- **触发器数据：** 触发自动化的记录中的字段（如客户名称、邮箱、订单金额）。
- **前序操作输出：** 来自之前步骤的数据，如新创建的记录 ID 或 AI 生成的文本。

包含变量的正文示例：

```
您好 {{客户名称}}，

您的订单 #{{订单号}} 已确认。

**合计：** ¥{{订单金额}}
**预计送达：** {{送达日期}}

感谢您的购买！
```

## 正文格式选项

| 格式 | 使用方式 | 适用场景 |
|---|---|---|
| 纯文本 | 直接输入消息内容 | 简单通知 |
| Markdown | 使用 Markdown 语法（`**粗体**`、`# 标题` 等） | 需要格式化的结构化消息 |
| HTML | 编写 HTML 标签（`<h1>`、`<p>`、`<table>` 等） | 自定义布局的精美邮件 |

你可以在同一正文中混合使用 Markdown 和动态变量。对于复杂的邮件设计，HTML 提供最大的灵活性。

## 附件处理

要包含附件：

- 引用前序步骤中的**附件字段**（例如上传到记录中的文件）。
- 附件将作为文件包含在发出的邮件中。
- 支持多个附件。

请注意，附件会增加邮件大小。较大的附件可能会因收件方邮件服务器的限制而导致投递问题。

## 适用场景

- **发送订单确认。** 新订单创建时，自动发送包含订单详情和感谢信息的邮件。
- **通知团队成员状态变更。** 任务变为"已阻塞"时，向负责人及其经理发送邮件。
- **发送定时报告。** 结合定时触发器和获取记录操作，发送每周摘要邮件。
- **发送个性化信息。** 使用记录数据为每位收件人自定义主题和正文。
- **确认表单提交。** 表单提交后，向提交者发送确认邮件，包含其提交的内容。

## 送达建议

- **使用自定义 SMTP**：如果发送高频量或业务关键邮件，建议使用自定义 SMTP。Teable 内置服务适合基本通知，但从您自己的域名发送的邮件更容易到达收件人的收件箱。
- **避免垃圾邮件风格的内容**：主题和正文中避免过多大写字母、感叹号或大量链接，这些可能触发垃圾邮件过滤器。
- **设置回复地址**：让收件人可以回复到真实的收件箱，而不是无回复地址。
- **控制附件大小。** 如需共享大文件，请改为包含下载链接。
- **正式启用前测试邮件**：使用测试数据手动运行自动化，验证格式和投递情况。
