Teable 应用构建器 (App Builder) 允许您在不编写代码的情况下，将数据库转化为功能齐全、定制化的 Web 应用程序。结合数据结构的力量和 AI 驱动的设计，您可以在几分钟内发布内部工具、客户门户和数据仪表板。

## 概览

应用构建器（App Builder）遵循 "Prompt to App"（提示词即应用）的工作流。您描述想要的内容，AI 为您构建应用结构、页面和逻辑。然后，您可以使用可视化编辑器或代码微调每一个细节。

> **应用构建器与 AI Chat 的区别**：应用构建器用于创建自定义界面和独立应用。如果在数据库内部进行交互式数据分析和临时提问，请使用 [AI Chat](/zh/basic/ai/ai-chat)。

## 创建应用

主要有两种创建应用的方式：

1. **从数据库创建**：在数据库中，点击 "+ 创建" 选择 "新应用" 生成一个链接到当前数据的应用。
2. **从对话创建**：打开 AI Chat 并说 "为这个客户表创建一个 CRM 应用"。

## 应用编辑器界面

应用构建器为非技术用户和开发者提供了双重界面。

### 1. 对话界面 (AI 驱动)

这是您的应用控制中心。
- **自然语言编辑**：描述您想做的更改，如 *"把标题改成蓝色"* 或 *"添加一个新记录提交表单"*。
- **智能路由**：AI 能够理解您指的是哪个组件或页面，并直接对其进行修改。

### 2. 实时预览面板

- **多设备模拟**：使用顶部的切换器在 **桌面**、**平板** 和 **移动** 视图之间切换。
- **即时反馈**：您的修改会立即渲染并显示。

### 3. 开发者模式 (代码编辑器)

为了实现无限的定制能力，我们提供了直接访问代码的权限。

- **Monaco Editor**：浏览器中类似 VS Code 的编码体验。
- **React 组件**：直接编辑底层 React 代码、Tailwind CSS 类和逻辑。
- **文件树**：完整浏览项目文件结构。

## 发布与管理

### 发布您的应用

当应用准备就绪时，点击右上角的 **发布** 按钮（箭头图标）。

- **公开链接**：Teable 会为您的应用生成一个安全的、可公开访问的 URL。
- **自定义域名** (即将推出)：支持绑定您自己的域名。

### 版本历史与回滚

Teable 自动跟踪每一次部署。

- 点击 **历史** 按钮（时钟图标）查看所有已部署的版本列表。
- **一键回滚**：如果新的更新出现问题，您可以即时将线上应用恢复到任何之前的稳定版本。

### 自动修复 (Auto-fix)

如果您的自定义代码导致构建错误，部署将会失败。

- Teable 在错误日志旁提供了一个 **Auto-fix (自动修复)** 按钮。
- 点击它，AI 将分析编译错误并自动修补代码以解决问题。

### 删除应用

要删除应用，请点击顶部的 **设置** (齿轮图标) 按钮，然后选择 **删除**。

## 最佳实践

- **由简入繁**：先让 AI 构建基础结构，然后迭代特定功能。
- **检查移动端**：频繁检查移动端预览，因为很多使用场景发生在手机上。
- **版本控制**：如果您想尝试不同的设计，可以保存版本或重置编辑。

### 必需配置

| 配置项 | 说明 |
|--------|------|
| **v0 API 密钥** | 从 [v0.app/chat/settings/keys](https://v0.app/chat/settings/keys) 获取 |

### 自定义域名 (可选)

应用构建器构建的应用部署在 Vercel 上。有两种方式使用自定义域名：

#### 方式 A：Vercel 域名 (自动配置)

从 Vercel 购买的域名会自动完成绑定配置。

1. 从 [Vercel Domains](https://vercel.com/domains) 购买域名
2. 在管理员后台的 **自定义域名** 字段中输入域名
3. 从 [Vercel 设置](https://vercel.com/account/settings/tokens) 获取并添加 **Vercel API Token**

#### 方式 B：外部域名 (手动 DNS 配置)

如果您已经从其他注册商 (如 Cloudflare、GoDaddy) 购买了域名：

1. 在 Teable 管理员后台，在 **自定义域名** 中输入您的根域名 (如 `example.com`)
2. 添加您的 **Vercel API Token**
3. 发布应用时，从 URL 中记录 **App ID** (如 `appXXX123`)
4. 在您的 DNS 服务商中为每个应用添加 CNAME 记录：

| 字段 | 值 |
|------|-----|
| 类型 | CNAME |
| 名称 | App ID (如 `appXXX123`) |
| 目标 | `cname.vercel-dns.com` |
| 代理 | 仅 DNS (如使用 Cloudflare) |

5. 在 Vercel 控制台 (项目 → 设置 → 域名) 中添加完整域名 (如 `appXXX123.example.com`) 并点击 **Verify**
6. 验证通过后，HTTPS 将自动配置

<Note>
每个应用会获得一个基于 App ID 的唯一子域名。不支持路径形式的 URL (如 `yourdomain.com/app/xxx`)。
</Note>

### API 代理 (可选)

对于访问 v0/Vercel API 受限的地区，可以配置代理 URL：

| 配置项 | 说明 |
|--------|------|
| **v0 Base URL** | v0 API 的代理地址 |
| **Vercel Base URL** | Vercel API 的代理地址 |

### AI Gateway (可选)

如果您使用 [Vercel AI Gateway](https://vercel.com/docs/ai-gateway)，可以在这里配置 API 密钥以启用：

- AI 请求的集中监控和日志记录
- 速率限制和成本控制
- 缓存以降低 API 成本

| 配置项 | 说明 |
|--------|------|
| **AI Gateway API Key** | 从 Vercel AI Gateway 设置中获取 |
